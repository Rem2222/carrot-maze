## 1. Кнопки на начальном экране с меню не работают

**Root cause:** Главное меню (main-menu overlay, строки 69-76) **не содержит HTML-кнопок** — только текстовые подсказки `Press ENTER to start`, `Press S for Settings` и т.д. Взаимодействие возможно только через клавиатуру.

**Детали:**
- `<div id="main-menu">` не имеет ни одного `<button>`
- ENTER обрабатывается через `handleKeyDown()` (строка 775), но **блокируется** `if(uiFocus) return` на строке 773 (uiFocus=true когда меню видимо)
- Резервный обработчик `starter()` (строки 868-870) не обрабатывает ENTER — он ловит только Arrow/WASD
- **Итог:** Нажатие ENTER на главном экране **не стартует игру**. Игра запускается только через клик по канвасу

**Объём:** ~30 строк — добавить 4 кнопки (Start, Settings, HOF, Instructions) + обработчики

## 2. При воскрешении после умирания игра не работает

**Root cause:** Функция `restartLevel()` (строки 652-660) не сбрасывает **бонусы и баффы** — `game.activeBuffs` и `game.bonusesOnField` остаются от предыдущей игры. Если перед смертью был активен `wall_reverse`, `pass_through`, `speed_up` или `slow_down`, они продолжают действовать после рестарта, ломая логику:

| Бафф | Эффект на restart |
|------|-------------------|
| `wall_reverse` | Змея умирает при входе на FLOOR (`updateSnake`: строка 729: `if (state.maze[nh.y][nh.x] === FLOOR) return false`) |
| `pass_through` | Змея проходит сквозь BED — бессмертие |
| `speed_up` / `slow_down` | Скорость искажена |

**Дополнительно:** `restartLevel()` не вызывает `clearBuffs()` — баффы висят.

**Доп. баг:** После `restartLevel()`, `game.dir = {dx:0, dy:0}` (нет направления). Пока игрок не нажмёт клавишу — змея стоит. В `handleKeyDown()` после рестарта `uiFocus=false`, но проверка `if(!game) return;` корректна. **Проблема не в обработке клавиш, а в активных баффах.**

**Объём:** +1 строка (`clearBuffs(game)`) в `restartLevel()`

## 3. Система бонусов неверная

### 3.1 Порядок открытия

**Текущий код:**
```js
slow_down:     unlockLvl: 1  // открывается с 1 уровня
speed_up:      unlockLvl: 1
plus_score:    unlockLvl: 1
kill_rabbit:   unlockLvl: 4
extra_life:    unlockLvl: 4
shorten_python:unlockLvl: 4
pass_through:  unlockLvl: 7
rage:          unlockLvl: 7
wall_reverse:  unlockLvl: 7
```

**Требование пользователя (из MUL-69):**
```
Уровень 3:  замедление (1 бонус)
Уровень 6:  + ускорение (2)
Уровень 9:  + убить зайца (3)
Уровень 12: + доп жизнь (4)
Уровень 15: + укоротить питона на 1/2 (5)
Уровень 18: + проход сквозь грядки (6)
Уровень 21: + озверин/rage (7)
Уровень 24: + реверс прозрачности (8)
```

**Проблемы:**
1. Бонусы открываются с 1 уровня (а надо с 3). Первый бонус — на 3 уровне
2. `plus_score` не упомянут пользователем — нежелательный бонус
3. `unlockLvl` не совпадает: 1,1,1,4,4,4,7,7,7 вместо 3,6,9,12,15,18,21,24

### 3.2 Отсутствует сообщение

При подборе бонуса не показывается сообщение "Бонус %получен". Только звук.

### 3.3 9-й бонус (нужно предложить)

**Рекомендация:** **Freeze (заморозка)** — зайцы застывают на 6 секунд (не двигаются, не прыгают). Открывается на 27 уровне. Синергия с охотой: змея спокойно съедает застывших зайцев. Иконка: ❄️, длительность: 6s.

**Объём:** ~40 строк — переименовать unlockLvl, убрать plus_score, добавить freeze, добавить сообщение

## 4. При воскрешении всё не сбрасывается + нет плавного перехода

### 4.1 Сброс при воскрешении

**Корень:** `restartLevel()` не сбрасывает:
- `game.bonusesOnField` — старые бонусы остаются на поле
- `game.activeBuffs` — старые эффекты продолжают действовать
- `game.bonusSpawnTimer` — таймер не сбрасывается

**Фикс:** Добавить `clearBuffs(game); game.bonusSpawnTimer = 0;` в `restartLevel()`

### 4.2 Уровневый переход (advanceLevel)

**Текущее поведение:** Мгновенный переход. Змейка телепортируется, зайцы перераспределяются. Нет визуальной обратной связи.

**Требование:** Показать "lvl N" с:
- Очки за прошлый уровень
- Всего очков
- Кол-во жизней
- Если на этом уровне открывается новый бонус — "Новый бонус: <name>"

**Объём:** ~40 строк — экран "lvl N" + таймер на 2-3 секунды

## 6. Кол-во зайцев для прохождения уровня — +1 каждые 5 уровней

**Root cause:** `RABBITS_PER_LEVEL = 3` (строка 308) — константа, никогда не меняется.

**Требование:** 
- Уровни 1-5: 3 зайца
- Уровни 6-10: 4 зайца
- Уровни 11-15: 5 зайцев
- И т.д.

**Формула:** `RABBITS_PER_LEVEL(level) = 3 + Math.floor((level - 1) / 5)`

**Объём:** 1 строка — заменить константу на функцию

## 7. При выходе из игры через меню — новая игра не начинается

**Root cause:** После `fullGameOver()` показывается game-over overlay (uiFocus=true). В `handleKeyDown()` (строка 773) `if(uiFocus) return;` блокирует все клавиши, включая ENTER. А game-over overlay не обрабатывает ENTER в `handleKeyDown()` — проверка `if(!game.alive && (e.key==="Enter"||e.key===" "))` на строке 780 есть, но она НИЖЕ `if(uiFocus) return;`.

**Цепочка:** Пользователь играет → ESC пауза → QUIT → main-menu → ENTER → `handleKeyDown` проверяет `uiFocus` (true, main-menu видимо) → return. ENTER не доходит до обработчика главного меню.

**Дополнительно:** В game-over overlay кнопка `PLAY AGAIN` работает (отдельный click handler).

**Фикс:** В `handleKeyDown()` — проверять main-menu и game-over ENTER до uiFocus guard.

**Объём:** 1-2 строки — переместить проверку main-menu/game-over ENTER до uiFocus guard.

## 8. Иногда невозможно съесть зайца

### Root cause 1: wall_reverse бафф убивает на FLOOR

Когда активен `wall_reverse` (строка 728-729):
```js
if (state.maze[nh.y][nh.x] === FLOOR) return false; // смерть на FLOOR!
```
Зайцы на BED достижимы с FLOOR. Но если заяц на FLOOR (floor rabbit, `isFloorRabbit=true`), подойти к нему невозможно — шаг на FLOOR убивает змею при wall_reverse.

### Root cause 2: Зайцы в состоянии eatingCarrot

Когда заяц ест морковку (`r.eatingCarrot = true`), он не уходит из обработчика `updateRabbits` (строки 706-707):
```js
if(r.eatingCarrot){
    if(r.carrotEatTimer<=0){r.eatingCarrot=false;...}
    continue;
}
```
В это время `r.jumping=false`, но заяц не меняет позицию — он продолжает сидеть на своей bedKey. **Это не баг** — змея может съесть его на следующем тике.

### Root cause 3: Зайцы на BED, окружённые BED

Наиболее вероятная причина: зайцы на BED-клетках, окружённых BED со всех сторон. Проверка `dist <= 1` (Manhattan) для съедения срабатывает, когда змея рядом с bed-клеткой зайца. Но если все соседние клетки зайца — BED (кроме той, где стоит змея), змея умрёт, пытаясь подойти. **В непроходимых лабиринтах (п.9) — гарантировано.**

**Объём:** ~5 строк — добавить safe-check в коллизию с зайцем

## 9. Лабиринты непроходимые (уровень 5 или 6)

**Root cause:** `generatePatterns()` (строки 584-591) имеет 3 варианта. Вариант 3 (33% шанс) — **концентрические кольца** — создаёт замкнутые BED-прямоугольники без проходов:

**Почему непроходимо:** Каждое кольцо — сплошной прямоугольник BED. Нет ни одного FLOOR-прохода. Змейка не может пересечь кольцо, зайцы внутри недостижимы.

**Фикс:** Убрать вариант 3 или добавить случайные проходы (1-2 FLOOR клетки на кольцо).

**Объём:** ~5 строк — модифицировать вариант 3, добавив random gaps
